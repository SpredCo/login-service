      General

      Information

      The login service of the Spred api allows registred clients to create user, and retreive access token.
      As explained in the RFC 6749, obtain an access token is needed to get resources from the api.

      Basic Authentication

      All requests sent to the login service MUST be authenticated by Basic Authentication. Here is the RFC
      Basically, you will need include in the HTTP Header of the request : 

      Authorization: Basic <CLIENT HASH>

      The <CLIENT HASH> field is the encoding in base 64 of your <client_key>:<client_secret>.
      If you don't have created a client yet, check the developer server documentation to create one.

      If a request is unauthenticated, the server will reply a status code 403 with a body Unauthorized.

      Tokens

      The login service will reply with an access object.
      The access_token will allow you to request the api, save it. But it have an expiration date, after it, the token won't work anymore and will have to refresh it.
      The refresh_token will allow you to refresh your access_token.
      The creation field tells if the user was created or not.

      Request and Reply format

      All requests must be HTTP request.
      Request body must be JSON formated.
      Reply are JSON formated object in snake_case.

      POST /v1/oauth2/token
      Basic Auth

      Get an access token

      This endpoint implements the OAuth2 exchange. Two grants are implemented: password (get token from user email and password) and refresh_token (get a new token from a refresh_token)

      Parameters

      Name Description Required In Schema
      grant_type

      Grant type, can be password (email and password fields will be required) or refresh_token (refresh_token field will be required)

      		 true body string
      username

      User email (required for password grant)

      		 false body string
      password

      User password (required for password grant)

      		 false body string
      refresh_token

      A refresh token (required for refresh_token grant)

      		 false body string

      Responses

      Http Status Code Description Schema
      200

      Access granted

      		 Access
      403

      Invalid grant (something went wrong with supplied params, check error message)

      		 Short Error
      400

      Invalid request (missing parameter)

      		 Short Error
